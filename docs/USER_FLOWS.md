# User Flows

## Founder Applicant Flow

1. Founder visits the landing page.
2. Founder clicks "Apply for Founder 12."
3. Founder creates account or logs in.
4. Founder completes the Founder 12 application.
5. System saves the venture/application as `submitted`.
6. AI analyzes the application.
7. AI generates:
   - Founder readiness score.
   - Founder 12 fit notes.
   - Role map.
   - 30-day roadmap.
   - Review questions for ISD.
8. Founder sees application status: under ISD review.
9. Admin reviews the application.
10. Admin marks the founder as shortlisted, accepted, or not selected.
11. If not selected, founder sees a clear status and optional next-step note.
12. If accepted, founder unlocks the Founder 12 member experience.

## Accepted Founder Flow

1. Accepted founder logs in.
2. Founder sees Founder 12 acceptance status.
3. Founder sees next Founders Coffee details.
4. Founder sees ISD member privileges.
5. Founder sees AI session information.
6. Founder sees external community chat link if configured.
7. Founder sees venture readiness, role needs, and roadmap.
8. ISD starts team formation when appropriate.
9. Admin assigns approved builders.
10. Founder sees assigned builders and missing roles.
11. Founder and team submit simple progress updates.
12. Venture moves toward prototype/demo readiness.

## Builder Flow

1. Builder visits landing page.
2. Builder clicks "Join the Builder Network."
3. Builder creates account or logs in.
4. Builder completes Builder Network profile.
5. Profile status becomes `pending`.
6. Admin reviews builder profile.
7. Admin approves or rejects builder.
8. If approved, builder becomes eligible for Founder 12 matching.
9. Admin assigns builder to an accepted Founder 12 venture.
10. Builder sees assigned venture only.
11. Builder contributes according to assigned role and availability.

## Admin Flow

1. Admin logs in.
2. Admin opens Cohort Command Center.
3. Admin reviews pending Founder 12 applications.
4. Admin reviews AI readiness score, role map, roadmap, and ISD review questions.
5. Admin shortlists promising founders.
6. Admin accepts up to 12 founders into Founder 12.
7. Admin marks other applicants as not selected or deferred.
8. Admin reviews pending Builder Network profiles.
9. Admin approves or rejects builders.
10. Admin opens team formation.
11. Admin reviews AI suggested builders for accepted ventures.
12. Admin manually assigns builders.
13. Admin updates venture/cohort status.
14. Admin tracks Founders Coffee readiness, AI session access, progress updates, and demo readiness.

## Founder 12 Selection Flow

1. Application submitted.
2. AI readiness report generated.
3. Admin reviews.
4. Admin shortlists.
5. Admin selects final 12.
6. Accepted founders unlock member area.
7. Not selected founders receive status and optional note.

## Team Formation Flow

1. Founder is accepted to Founder 12.
2. AI role map identifies needed roles.
3. System compares approved builders against role needs.
4. System creates suggested matches.
5. Admin reviews suggestions.
6. Admin assigns one or more builders.
7. Assigned builders appear on venture page.
8. Missing roles remain visible until filled.

## Founder Status Flow

1. Application Submitted
2. Under ISD Review
3. Shortlisted
4. Accepted To Founder 12
5. Not Selected For This Cohort
6. Team Formation
7. Building
8. Demo Ready
9. Alumni
10. Archived
