# Concept Docs Index

This index explains the first concept document set for La Mesa.

Read these before changing product scope, database design, user flows, or production implementation plans.

2. `PRODUCT_STRATEGY.md`

3. `BATCH_01_PLAYBOOK.md`

4. `FOUNDER_READINESS_MODEL.md`

5. `BUILDER_NETWORK_MODEL.md`
   Defines the La Mesa Builder Network, approval model, matching criteria, and reliability signals.

6. `TRACKS_AND_TABLES_MODEL.md`

7. `ANTI_GOALS_AND_PRODUCT_PRINCIPLES.md`
   Defines what La Mesa must not become and the principles that should guide product decisions.

## Locked Concept Decisions

- La Mesa is a selective AI-assisted startup fellowship.
- Founders can create accounts directly, but they are vetted only after project submission.
- Founder state language should say "pending consideration for La Mesa Summer 2026 Table."
- Builders apply to join the La Mesa Builder Network.
- Builders cannot see projects before approval.
- Public/brand term: Table.
- Internal/startup term: Batch.
- System name: La Mesa Batch 01.
- Participant-facing identity: Table 01.
- Seasonal public name: La Mesa Summer 2026 Table.
- Founder scoring should be visible to founders as a Founder Readiness Report.
- Builder reliability should become visible over time.
- Tracks should have their own identity and cohort experience.
- AI recommends. ISD decides.
